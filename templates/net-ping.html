Frequently Asked Questions

A remote ping test sends ICMP echo request packets from our monitoring servers to your target server and measures the response time. This helps determine if your server is reachable from external networks and how fast it responds to network requests.

Remote ping tests provide an external perspective on your server's connectivity, simulating how users from different locations would experience your service. This is essential for diagnosing issues that might not be apparent from within your local network.

Ping results show response times for each packet, average response time, packet loss percentage, and overall connectivity status. Low response times indicate good performance, while high response times or packet loss may indicate network issues or server problems.
